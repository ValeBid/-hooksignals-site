# Agent 02 — Product Trust Auditor

## Purpose
Find and eliminate every trust-killing issue in the product before it reaches real users. A single fake claim, broken auth flow, or confusing pricing can permanently lose a potential customer.

## When to Run
- Before any new public page launches
- Before any pricing or conversion change
- Monthly trust review
- After any content update

## Inputs
- Live production URL
- All public-facing pages
- All tool output samples

## Checks

### FAKE CLAIMS — severity: BLOCKER
- [ ] No fake creator names ("10,000 creators use HookSignals", "top YouTubers love it")
- [ ] No fake revenue claims ("$10M in creator revenue boosted")
- [ ] No fake testimonials with invented names or quotes
- [ ] No "guaranteed viral" or "guaranteed results" language
- [ ] No "live" or "real-time" metrics that are hardcoded
- [ ] Any demo/sample outputs are clearly labeled: "Example output", "Sample analysis"
- [ ] No "trusted by X creators" unless X is a verified, accurate number

### AUTH TRUST — severity: BLOCKER
- [ ] Sign-in page renders the actual Clerk widget (not blank)
- [ ] Sign-up page renders the actual Clerk widget (not blank)
- [ ] "Free to start" claim is accurate (user actually gets free credits)
- [ ] Credit count on sign-up matches what is actually credited in the database
- [ ] Dashboard shows real user data, not hardcoded demo data

### PAYMENT TRUST — severity: BLOCKER
- [ ] Pricing page shows accurate prices (matches Paddle price IDs)
- [ ] Checkout uses real Paddle integration (not mocked)
- [ ] "Cancel anytime" claim is accurate (Paddle allows it)
- [ ] Trial language is accurate ("7-day free trial if enabled" is honest)
- [ ] Refund policy page exists and is linked from checkout/pricing

### PRODUCT HONESTY — severity: HIGH
- [ ] Tools that are "coming soon" are clearly labeled as such
- [ ] YouTube Video Analyzer shows "Early Access" — not a broken form
- [ ] Tool outputs are generated by actual AI, not hardcoded responses
- [ ] Credit cost per analysis is shown before the user submits
- [ ] Empty states show helpful content, not broken/blank UI

### LEGAL AND TRUST PAGES — severity: HIGH
- [ ] Privacy policy exists at `/privacy`
- [ ] Terms of service exists at `/terms`
- [ ] Refund policy exists at `/refund-policy`
- [ ] Support email is correct: `support@hooksignals.com`
- [ ] Copyright year is current

### VISUAL TRUST — severity: MEDIUM
- [ ] No stock photos of fake creators
- [ ] No placeholder "Lorem ipsum" text in production
- [ ] Pricing cards match what the database actually grants
- [ ] No "Award-winning" or "Industry-leading" without evidence

## Scoring Rubric
- BLOCKER: Must fix before any user-facing launch
- HIGH: Fix within same sprint
- MEDIUM: Fix within 2 weeks

## Output Format
```
TRUST AUDIT REPORT
Date: YYYY-MM-DD
Auditor: <name>

FAKE CLAIMS: [PASS/FAIL] — <details>
AUTH TRUST: [PASS/FAIL] — <details>

LEGAL PAGES: [PASS/FAIL] — <details>

VERDICT: TRUSTWORTHY / NEEDS FIXES
Issues found: <count>
Blockers: <list>
```
